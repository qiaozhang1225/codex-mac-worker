# Skill-First Dual-Mac Collaboration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Retire the unattended GitHub-App Worker and replace it with one versioned Codex skill that lets the MacBook dispatch complete GitHub Issue contracts and the visible Mac mini Codex App execute, checkpoint, and deliver them safely.

**Architecture:** The repository becomes skill-first: a concise `SKILL.md` routes role judgment to four focused references and invokes deterministic Python helpers for Issue validation/writes and guarded Git delivery. Both Macs install the same repository commit; GitHub Issue bodies are the sole current task contract, while comments contain start/checkpoint/delivery evidence. The legacy daemon, SQLite state machine, GitHub App gates, PR/Ruleset assumptions, and `codexctl` compatibility layer are removed as one migration.

**Tech Stack:** Codex skills, Python 3.12, PyYAML 6, `gh` CLI, Git/worktrees, pytest, macOS shell scripts, SSH for Mac mini operations.

## Global Constraints

- Do not use Codex Goal/“目标” mode or `codex exec` as a task runner.
- Do not retain a daemon, Issue poller, SQLite runtime state, GitHub App identity gate, readiness attestation, mandatory PR, Ruleset profile, assisted merge, or automatic merge compatibility mode.
- MacBook remains the only dispatcher; user confirmation is required before each formal Issue creation.
- Mac mini may execute implementation details inside an approved contract but may not make product decisions, widen scope, change delivery mode, deploy, access production data, or delegate again.
- Estimated duration is not a delegation criterion. Discussion maturity, product-decision closure, and a complete continuous plan are required.
- Issue body is the sole current task contract. Every task-contract edit increments `revision`; comments cannot change the contract.
- `delivery_mode` is `direct-main` by default and `task-branch` when selected by MacBook. Never force push.
- Every implementation task follows RED-GREEN-REFACTOR. Every skill behavior is baseline-tested without the skill before authoring and forward-tested with the skill before deployment.
- Preserve user work. Do not edit the dirty EaseWise checkout at `/Users/qiaoz-macair/Documents/Easewise`; use an isolated worktree during Task 8.
- Do not transmit or print the GitHub App private key. Isolate it only after the legacy service is stopped.
